- The badge should be a **pill/rounded tag** with bold uppercase text

## 📍 PART 2 — Plan Selection on "Create New Restaurant / Menu" Page

When a user creates a new restaurant or menu, add a **Plan Selection Section** as a required step in the form/flow.

### UI Requirements:
- Show all 3 plan cards side by side (or stacked on mobile)
- Each plan card must display:
  - Plan name (PLUS / PRO / MAX) in bold uppercase
  - Plan color/icon (🟦 / ⬜ / 🟨)
  - A short feature list (use placeholder features for now):
    - PLUS: "Up to 2 Menus, Basic Analytics, Standard Support"
    - PRO: "Up to 10 Menus, Advanced Analytics, Priority Support"
    - MAX: "Unlimited Menus, Full Analytics, Dedicated Support, Custom Branding"
  - A "Select Plan" button

### Interaction:
- Clicking a plan card **selects** it (highlighted border/glow matching plan color)
- Only ONE plan can be selected at a time
- Selected plan is stored in form state: `selectedPlan: "PLUS" | "PRO" | "MAX"`
- The selected plan value must be saved when the restaurant/menu is created
- Default selected plan: **PLUS**

### Visual:
- Selected card: glowing border in the plan's color + checkmark icon ✓ in top-right
- Unselected cards: muted border, slightly faded
- Smooth transition animation on selection

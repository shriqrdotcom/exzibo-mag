==================================================
CONFIRMED PROBLEMS
==================================================

Repository evidence indicates:

1. server.js can skip authentication when:

   DISABLE_AUTH=true

   or:

   VITE_DISABLE_AUTH=true

2. Some development branches fabricate or assume privileged identities, including owner or superadmin authority.

3. Deployable Express or Replit execution can therefore become unauthenticated through environment-variable misconfiguration.

4. Better Auth secret validation is mainly tied to Vercel-specific runtime detection.

5. Other production-capable runtimes may use a randomly generated per-process Better Auth secret.

6. A random runtime secret:

   - hides missing configuration;
   - invalidates sessions after restart;
   - produces inconsistent behaviour between instances;
   - is not acceptable for production.

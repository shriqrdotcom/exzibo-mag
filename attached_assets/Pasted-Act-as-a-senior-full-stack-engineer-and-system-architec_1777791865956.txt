Requirement:

1. Customer places order from MENU.

2. Order must instantly appear in:
   
   - MASTER
   - ADMIN
   - MANAGER
   - EMPLOYEE dashboards

3. Each dashboard has:
   
   - CONFIRM button
   - CANCEL button

4. When CONFIRM or CANCEL is clicked:
   
   - Order status updates in database
   - Change must reflect instantly on:
     → MENU "Orders Page"

Implementation Logic:

- Create "orders" table:
  fields:
  id
  restaurant_uid
  items
  status (pending / confirmed / cancelled)
  created_at

- MENU:
  
  - Insert order with status = "pending"

- DASHBOARDS:
  
  - Subscribe to orders table
  - Show live incoming orders

- ACTIONS:
  
  - On CONFIRM → update status = confirmed
  - On CANCEL → update status = cancelled

- MENU Orders Page:
  
  - Subscribe to same order
  - Reflect live status updates

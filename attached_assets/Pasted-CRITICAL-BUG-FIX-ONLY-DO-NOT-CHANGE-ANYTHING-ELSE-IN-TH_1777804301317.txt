STEP 3 — IMPLEMENT REAL-TIME LISTENER:

IF the project uses SUPABASE:

Replace the one-time fetch on the order tracking page with a
Supabase real-time channel subscription. Use this exact pattern
(adapt table/column names to match what already exists in the project):

  useEffect(() => {
    // Initial fetch
    fetchOrderData();

    // Real-time listener
    const channel = supabase
      .channel('order-status-' + orderId)
      .on('postgres_changes', {
        event: 'UPDATE',
        schema: 'public',
        table: 'orders',        // use actual table name in project
        filter: `id=eq.${orderId}`
      }, (payload) => {
        setOrder(payload.new);  // update local order state instantly
      })
      .subscribe();

    return () => supabase.removeChannel(channel);
  }, [orderId]);

IF the project uses FIREBASE FIRESTORE:

Replace getDocs/getDoc with onSnapshot() so it listens live:

  useEffect(() => {
    const unsubscribe = onSnapshot(
      doc(db, 'orders', orderId),
      (docSnap) => {
        if (docSnap.exists()) {
          setOrder({ id: docSnap.id, ...docSnap.data() });
        }
      }
    );
    return () => unsubscribe();
  }, [orderId]);

IF the project uses REST API / custom backend:

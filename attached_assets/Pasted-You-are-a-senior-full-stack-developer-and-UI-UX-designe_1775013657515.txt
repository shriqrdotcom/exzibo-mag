Your task is to build a production-level feature for a SaaS restaurant admin dashboard.

-----------------------------------
🔷 CORE FUNCTIONALITY
-----------------------------------

When a Super Admin creates a new restaurant using the "Create Restaurant Form":

1. A unique restaurant website should be automatically generated.
2. Assign a unique URL for each restaurant like:
   /restaurant/{restaurantSlug}

3. The "Visit Your Website" button on the dashboard MUST:
   - Dynamically link to that restaurant’s live website
   - Open the generated restaurant page
   - Work without reload errors or broken routing

-----------------------------------
🔷 TECH REQUIREMENTS
-----------------------------------

- Use React Router (or Next routing if Next.js)
- Store restaurant data in a simple state/store (or mock DB)
- Each restaurant must have:
  - name
  - slug (auto-generated from name)
  - images (array)
  - menu items
  - description

-----------------------------------
🔷 ROUTING LOGIC
-----------------------------------

- After form submission:
   → redirect to admin restaurant dashboard page

- "Visit Your Website" button:
   → should route to: /restaurant/{slug}

- Ensure:
   ✔ No blank page
   ✔ No console errors
   ✔ Works on refresh
